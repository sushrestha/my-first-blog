This challenge introduces the basics of SQL Injection.

SQL(Standard Query Language): Is a language that is used to communciate to a database.
It is commonly used in websites when logging in as a user. A typical query to log in
jonDoe with 123456 for his password to a website may look like this:


SELECT * FROM Users WHERE username='jonDoe' AND password='123456';



Some websites don't do any sanitization or escaping of characters when taking user input.
An attacker to see if sql injection can work would try somethign similiar to how you
solved the puzzle. The reason behind it is because you are giving malformed input that
shouldn't be allowed with out some changes to it. With no sanitization the same query from
above changes to something like this:


SELECT * FROM Users WHERE username=' ' OR 1=1; -- AND password='123456';


This new SQL statment is true for every single user and prints their information out,
because of two reasons.
1: The addition of OR 1=1 makes the statment always true for every single user entry
2: The new end of the statment is at the ; right after 1=1, and the -- comments out the
rest the statment( AND password='123456';).
